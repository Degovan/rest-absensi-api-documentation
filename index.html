CATATAN

  • Penambahan kolom address di tabel attendance
    alasan
    • karena di mobile terdapat input yaitu address, sedangkan di tabel attendance (absen) tidak terdapat kolom yang menampung data tersebut, akhirnya saya tambahkan kolom address pada tabel tersebut
    • Untuk tipe data di kolom address adalah ( text, nullable )
  • Pengubahan tipe data pada kolom renke yang awal nya adalah varchar menjadi longtext
    alasan
    • di kode program sebelumnya untuk menghandle data array rencana_kerja dan status, itu menggunakan perulangan lalu insert ke database. sehingga nantinya akan menghasilkan beberapa row data kolom yang memiliki nilai sama, hanya berbeda di kolom renke dan status
    • ini berpengaruh di database nya sendiri, karena akan terjadi duplikasi data dan memenuhi kapasitas database
    • agar kode dan database lebih efektif, jadi oleh saya kolom renke diubah menjadi tipe data longtext
    • nanti untuk proses insert ke kolom renke, langsung saja saya parse ke json setelah itu diinput
    • ini juga nantinya memudahkan orang bagian front end, karena untuk mengambil data renke hanya perlu melakukan perulangan data kolom, bukan perulangan data row
  • menemukan kolom yang tidak digunakan yaitu status
    • alasan
    • nah karena hal diatas, untuk rencana_kerja dan status. Oleh saya dibuat satu kesatuan array, yang diinsert ke kolom renke
    • sehingga saya rasa kolom status tidak digunakan
  • di tabel harian ( rencana_kerja ), untuk kolom lokasi_id dan jabker_id. Saya isi data statis dikarenakan di tabel employees tidak memiliki kolom yang ber relasi dengan tabel lokasi dan jabker
